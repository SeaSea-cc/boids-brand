## 1. Brand Positioning

### 1.1 What Boids Is

Boids is an expert-agent platform for building, distributing, and working with AI agents.

Boids ships as two distinct products serving two distinct audiences:

**Boids product** is a consumer-facing app for regular users. It lets anyone work with expert agents — inside the Boids interface or integrated into the tools they already use, like Slack and Telegram.

**Boids API / CLI** is developer infrastructure. It lets developers build agent-powered products, integrate expert agents into their own applications, and distribute through the Boids network.

Both products are built on the same agent network. What is published by one is callable by the other.

### 1.2 What Boids Is NOT

| Not This | Why |
|----------|-----|
| "AI chatbot marketplace" | Too consumer, too passive |
| "Talk to AI celebrities" | Persona-agent framing, not professional work |
| "No-code AI tool" | Tool framing; Boids is a platform |
| "Developer API marketplace" | Too narrow; ignores the consumer product |
| "AI app store" | Too static; agents on Boids are built, distributed, called, composed, and used |

### 1.4 Two Product Lines

---

**Boids Product — For regular users.**

The Boids consumer product is for anyone who wants to work with expert agents. No technical knowledge required.

Users access expert agents directly inside the Boids app, or integrate them into the tools they already use — Slack, Telegram, and other IM platforms. Agents become callable coworkers that live where the work happens, not in a separate app.

Key use cases:
- Use expert agents inside the Boids product
- Integrate agents into Slack, Telegram, and IM tools
- Call specialist agents for specific tasks without switching context
- Build and publish agents through the product UI without technical setup

Positioning:
> Bring expert agents to work with you.

---

**Boids API / CLI — For developers.**

The Boids API and CLI are developer infrastructure. Developers use them to build agent-powered products, integrate the agent network into their own applications, and distribute through A2A interfaces.

Developers don't have to build the agent layer themselves. Boids provides the infrastructure — the distribution network, the credit loop, the A2A interfaces — so developers can focus on their product.

Key use cases:
- Build agents programmatically via API and CLI
- Integrate expert agents into apps and products
- Distribute through the Boids network and A2A interfaces
- Compose agents into multi-agent workflows

Positioning:
> Build products with agents. Ship through API and CLI.

---

### 1.5 The Agent Network

Both product lines share the same underlying agent network. Agents built and published through either product are callable across the entire platform — by users in the consumer product, by developers via API, and by other agents via A2A.

Every Boids agent is:

| Value | Meaning |
|-------|---------|
| Callable | Usable through the product, API, CLI, and A2A |
| Composable | Works with other agents inside larger tasks |
| Distributable | Built once, brought into many contexts |
| Verifiable | Ranked, credited, and trusted through usage signals |
| Work-native | Designed for real work, not just chat |

**Canonical sentence:**
Boids is made for A2A: expert agents that can be called, composed, and distributed across human and agent workflows.

### 3.4 Narrative Arc (Six Beats)

**Pain**
Most AI resets every session. Specialized expertise doesn't travel. Developers rebuild agent infrastructure from scratch. Regular users switch between apps to access AI instead of AI coming to them.

**Problem**
There's no standard layer for building agents, publishing them, and getting them into the places where work actually happens — neither for the people who build products nor for the people who just want to get work done.

**Reveal**
Boids is the expert agent platform — a consumer product where regular users bring expert agents into Slack, Telegram, and their daily work, and a developer platform where teams build agent-powered products via API and CLI. One shared agent network underneath.

**Proof**
Regular users integrate expert agents into Slack and Telegram in minutes — no setup, no switching apps. Developers build on the API and distribute through the network without rebuilding the agent layer. The same agents, callable by both.

**Outcome**
Professional expertise becomes callable infrastructure. Developers ship faster. Users work with specialists that live where the work already happens. The agent network compounds with every new agent published.

**CTA**
Bring an expert agent to work — or build the next one.

## 4. Visual Identity

### 4.1 Color System — "Precision Mono + Ambient Glow"

A strict black-and-white base with a three-color ambient glow system as the decorative layer. Black and white carry the structure; the glow carries the energy. The decorative layer is never a stripe gradient — it is a radial, atmospheric wash. In dark mode: vibrant glows on deep black. In light mode: soft pastel washes on white. Both use the same color family.

#### Base Palette

| Role | Name | Hex | Use |
|------|------|-----|-----|
| Brand Primary | Black | #0D0D0D | Hero surfaces, primary text, brand-mark fill |
| Light Surface | White | #FFFFFF | Workspace canvas, content backgrounds |
| Neutral 50 | Gray-50 | #F7F7F7 | Card surfaces on White |
| Neutral 400 | Gray-400 | #9E9E9E | Secondary text, metadata, captions |
| Neutral 700 | Gray-700 | #3D3D3D | Body text, hairlines |
| Deep Surface | Onyx | #1A1916 | Dark-mode surface, terminal frames |
| Functional Warning | Ochre | #C99B3F | Retry / pending — never decorative |
| Functional Error | Terracotta | #C24A3A | Build failure — never decorative |

#### Ambient Glow — Three-Color System

Three colors used as radial ambient glows — never as stripe gradients. In dark mode: vibrant, saturated glows on deep black. In light mode: soft pastel washes on white. Same three hues, different intensity.

```
#06B6D4   Cyan / Deep Blue    — cool, technical, active
#8B5CF6   Purple / Violet     — depth, intelligence, brand
#EC4899   Pink / Rose         — warmth, energy, human
```

**Glow usage rules:**

| Context | Usage |
|---------|-------|
| CTA button border | 1–2px glow gradient stroke, Black fill inside |
| Active agent nodes / selected state | Glow via box-shadow |
| Brand moments (cover, hero) | Radial glow as background light source |
| Icon accent | Glow color stroke, 1.5px weight |
| Credit indicators / agent distribution state | Glow pulse animation |

**Hard rules:**
- Never fill large background surfaces with glow colors directly.
- Glow is always radial, diffused, atmospheric — never a sharp stripe or flat fill.
- On dark (Black/Onyx): full saturation. On White: desaturated, pastel tint.
- All three colors must coexist on a surface — no single-color pulls.

#### Base Hard Rules

- No cool gray. Banned: #FAFAFA, #F5F5F5, #E5E5E5.
- Tinted shadows only: Black at 8% opacity, never rgba(0,0,0,*).
- Texture mandate: every brand surface carries grain overlay at 3% opacity, pointer-events: none.

### 4.2 Typography — "Builder Stack"

| Level | Family | Weights | Use |
|-------|--------|---------|-----|
| Display | PP Editorial New | Ultralight (200), Italic | Hero headlines, brand moments |
| Display alt | Cabinet Grotesk | Bold (700), Extrabold (800) | Eyebrow tags, section titles, deck headers |
| Body | Geist | Regular (400), Medium (500), Semibold (600) | Product UI, documentation, body copy |
| Mono / Data | JetBrains Mono | Regular (400), Medium (500) | Agent logs, API configs, credit tables |

**Hard rules:**
- **Banned:** Inter, Roboto, Arial, Open Sans, Helvetica, SF Pro, Source Sans Pro, Space Grotesk.
- Tracking: display = -0.03em. Eyebrow tags = +0.2em uppercase 10px.
- Title-to-subtitle ratio ≥ 3.0×.
- Tabular nums on all data: `font-variant-numeric: tabular-nums`.
- Italic Editorial New for single-word emphasis. Never bold for emphasis.
- Sentence case only. No Title Case headers.
- Numbers always in JetBrains Mono or Geist tabular-nums.

### 4.3 Surface & Material — "Double-Bezel"

Every card, panel, or major container uses nested-enclosure construction:

```
Outer Shell:  bg = Gray-50,  padding = 6px,  ring = 1px Black @ 5%,  radius = 24px
Inner Core:   bg = White,    padding = 24px,                           radius = 18px
              + inset highlight: shadow inset 0 1px 1px White @ 60%
```

**Primary CTA button:** pill (rounded-full), padding 12px 24px, fill = Black, text = White, glow border (1.5px gradient stroke). Trailing arrow ↗ inside its own 32×32 circle (glow @ 12%).

```
[ Build your first expert agent   ( ↗ ) ]
```

**Eyebrow tag:** Cabinet Grotesk 10px, +0.2em uppercase, bg: glow @ 10%, text: Black.

```
〔 EXPERT AGENT PLATFORM 〕
Build expert agents. Bring them everywhere.
```

**Texture & shadow:**

| Element | Spec |
|---------|------|
| Grain overlay | Fixed, inset: 0, pointer-events: none, opacity: 0.03 |
| Card shadow | 0 1px 2px Black/0.04, 0 8px 24px Black/0.06 |
| Hover lift | translateY(-2px) + shadow → Black/0.10, cubic-bezier(0.32, 0.72, 0, 1) 400ms |
| Banned | shadow-md, rgba(0,0,0,0.3), glassmorphism on scrolling content |

### 4.4 Layout System

| Archetype | Use For | Spec |
|-----------|---------|------|
| Asymmetric Bento | Marketing pages, product showcases | CSS Grid, varied col-span. Never 3-equal-card row. |
| Editorial Split | Narrative landing, About, Manifesto | w-1/2 Editorial New left + scrollable interactive right. |
| Document Bento | Docs, API reference, agent configs | Max-width 1200px, asymmetric grid, 65-char body |

**Universal rules:**
- Section padding: py-32 desktop / py-16 mobile minimum.
- Whitespace ≥ 40% per surface.
- Max content width: 1200px–1440px. Never edge-to-edge.
- Body copy: 65ch maximum.
- `min-height: 100dvh` for full-screen sections.
- Mobile (<768px): single column, w-full, px-4.

### 4.5 Iconography

| Tier | System | Use |
|------|--------|-----|
| Primary | Phosphor Icons (Light, 1.5px stroke) | All UI iconography |
| Brand | Custom Boids glyphs | Build / Distribute / Call / Compose / Credit |
| Banned | Lucide, Feather, FontAwesome, Material Icons | Generic defaults |

One stroke weight: 1.5px across all surfaces.

### 4.6 Forbidden Visuals

**Always banned:**
- Emojis in brand contexts
- Cartoon / 3D-glossy "AI brains" or robots
- Stock-photo people
- Handshake / coin / rocket / flag / globe metaphors
- Centered all-caps body text
- Isolated chat-input screenshots
- Equal-height 3-card feature rows

**Color / surface bans:**
- Glow colors as full background fills
- Teal-to-blue or purple-to-blue stripe gradients
- Glassmorphism on scrolling content
- Cool grays (#FAFAFA, #F5F5F5, #E5E5E5)
- Generic box-shadow: rgba(0,0,0,0.1)
- Centered hero with ghost-button-pair below H1

## 5. Application Guidelines

### 5.1 Slide / Deck Layout (1920×1080)

- 5% top-left blank square reserved (logo safe zone).
- Bottom-right `Privileged&Confidential` watermark: 10px JetBrains Mono, Black @ 35%.
- Whitespace ≥ 40%.
- One core insight per slide. Never two H1s.
- Footer source citation: 9px JetBrains Mono, Gray-400, left-aligned.
- Grain overlay mandatory at 3% on every slide.
- Page numbers: JetBrains Mono, Gray-400, top-right, every slide except cover.
- Glow: radial ambient light source only — never stripe wash across slide.

### 5.2 GPT Image 2 Prompt Preamble

**Required Elements:**

| Element | Phrase that must appear |
|---------|------------------------|
| Aspect ratio | 16:9 AND 1920×1080 |
| Visual register | systems-thinking, builder-grade visual standard |
| Background | White (#FFFFFF) or Black (#0D0D0D) only |
| Texture | 3% film-grain texture |
| Accent | Ambient glow using #06B6D4, #8B5CF6, #EC4899 — radial, diffused, never stripe |
| Banned colors | No flat teal fills, no stripe gradients, no single-color glow pulls |
| Banned fonts | Never Inter, Roboto, Arial, Open Sans, Helvetica, Space Grotesk |
| Required fonts | PP Editorial New AND Cabinet Grotesk AND Geist AND JetBrains Mono |
| Surface system | Double-Bezel nested-enclosure architecture |
| Shadow rule | Black-tinted at low opacity, never rgba(0,0,0,*) |
| Logo safe zone | 5% top-left blank square, no logo |
| Logo prohibition | Do not draw the Boids logo |
| Watermark | Privileged&Confidential (exact spelling, no space) |
| Page number | Top-right, small, JetBrains Mono |

**Prompt Block Structure:**

```
### Page NN Prompt — <Title>

Slide theme: "<Title> — <Hook Sentence>."

Core content: <Specific content. Numbers, named modules, named agent capabilities. No vague hype.>

Visual direction: <Layout archetype. Glow placement (radial, atmospheric, never stripe). Restate forbidden visuals.>
```

### 5.3 Logo Rules

- Wordmark in Cabinet Grotesk Extrabold, -0.04em tracking.
- Approved surfaces: Black, White, Onyx — with ambient glow as background accent only.
- Never on: gradient fills, busy photography, low-contrast surfaces.
- With tagline: Geist Medium at 60% wordmark size, +0.05em tracking, Gray-400.
- In GPT Image 2 prompts: never draw the logo; reserve the 5% top-left square.
